# ADR — Платформа для краудфандинга

Этот каталог содержит Architecture Decision Records (ADR) для проекта **«Платформа для Краудфандинга»**.

## Как читать ADR
Каждый ADR описывает:
- **Контекст** (почему возникла необходимость решения),
- **Решение** (что выбираем),
- **Альтернативы** (что рассматривали),
- **Последствия** (плюсы/минусы и влияние на систему).

## Индексация
Файлы называются по шаблону `ADR-XXXX-<short-title>.md`, где `XXXX` — порядковый номер.

## Статусы
- Proposed — предложено, не утверждено
- Accepted — принято
- Deprecated — не использовать в новых решениях
- Superseded — заменено другим ADR
